DLCore

Kernel funcional sobre el contexto de ejecución de DLRoute. ORM, plantillas, autenticación, .env.type y bootstrap con Project::run() — sin reinventar la infraestructura HTTP.

DLCore no redefine el entorno: lo consume de forma determinista y delega rutas y servidor a DLRoute.

Kernel

Responsabilidades del kernel

Qué hace DLCore en el stack

  • Consumir el contexto de servidor sin duplicar resolución de rutas base.
  • Orquestar bootstrap, autoload de rutas y ciclo de vida de la aplicación.
  • Exponer APIs estables para datos, vistas, autenticación y configuración.
  • Integrar DLStorage para contenedores .dlstorage y entropía de credenciales.

Onboarding

Inicio rápido

Cinco pasos para integrar dlunire/dlcore, arrancar el bootstrap y tener una ruta respondiendo.

  1. 1

    Requerir el kernel

    En un proyecto PHP con Composer, declare dlunire/dlcore. Composer resuelve dlroute y dlstorage como dependencias transitivas.

    terminal 
    $ composer require dlunire/dlcore

    Alternativa en composer.json: "dlunire/dlcore": "^2.1" y luego composer update.

  2. 2

    Configurar el entorno

    Archivo .env.type en la raíz: cada variable tiene tipo estático. Con un mínimo basta para arrancar sin base de datos. Las variables DL_DATABASE_* y DL_PREFIX solo hacen falta si va a usar el ORM.

    .env.type · mínimo 
    DL_PRODUCTION: boolean = falseDL_LIFETIME: integer = 3800

    Opcional: correo, reCAPTCHA, CORS (DL_CORS_ORIGINS). Detalle en el capítulo 2.

  3. 3

    Punto de entrada

    En public/index.php: autoload de Composer y Project::run(). Use DIRECTORY_SEPARATOR para rutas portables (Windows y Unix).

    public/index.php 
    <?php use DLCore\Boot\Project; require_once dirname(__DIR__). DIRECTORY_SEPARATOR . 'vendor'. DIRECTORY_SEPARATOR . 'autoload.php'; Project::run();
  4. 4

    Registrar una ruta

    En routes/web.php — se cargan al arrancar con Project::run() (salvo autoload_routes: false).

    routes/web.php 
    <?php use DLRoute\Requests\DLRoute; DLRoute::get('/api/health', fn () => ['status' => 'ok',]);

    Modelo y convención de tablas

    Opcional en este paso: un modelo vacío y el prefijo ORM.

    app/Models/Products.php 
    <?phpnamespace App\Models; use DLCore\Database\Model; final class Products extends Model {}

    Con DL_PREFIX: string = "dl_"

    Products → dl_products

    Plantillas en resources/*.template.html

  5. 5

    Levantar el servidor

    Document root en public/. Servidor embebido de PHP:

    terminal 
    $ php -S localhost:8000 -t public/# → http://localhost:8000/

    Pruebe GET /api/health. Si usa el skeleton completo (dlunire/dlunire), también puede composer run dev en el puerto 3000.
